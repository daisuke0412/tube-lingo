## 画面遷移図

```
[S-01 初期画面]  /
    │
    │ 開始ボタン押下（URLバリデーション通過）
    │ → navigate("/learn/{videoId}")
    │
    └──→ [S-02 学習画面]  /learn/:videoId
              │
              ├─ マウント時に /api/transcript を呼び出し
              │   ├─ 取得中 → ローディング表示
              │   ├─ 成功 → 字幕リスト表示
              │   └─ 失敗 → エラー表示 + 「戻る」ボタン
              │
              ├─ [AIに聞く] / 💬タップ ──→ [S-02-M AIチャットモーダル]
              │                                    │
              │                          ✕ボタン ──┘（S-02に戻る）
              │
              └─ APIエラー ──→ 各画面内にインライン表示
```

**ルーティング方式の特徴:**
- `/learn/AJpK3YTTKZ4` のようなURLでブックマーク・共有が可能
- 字幕取得は学習画面のマウント時に実行（初期画面では行わない）
- APIキーは学習画面内で必要に応じて入力（AIチャットモーダル内）

## 各画面の共通フォーマット

各画面設計書は以下のセクションで構成する。

| セクション | 内容 |
|---|---|
| ヘッダー表 | 画面ID / 画面名 / 表示条件 / 関連機能 |
| モックアップ | ASCIIアートによるUI構成。状態ごとに複数記載可 |
| 入力バリデーション | フィールドごとのルール。なければ「なし」と記載 |
| 画面イベント | 実行条件・処理内容・備考（初期表示を必ず含む） |
