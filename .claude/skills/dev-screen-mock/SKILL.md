# 画面モック実装スキル

## 概要
画面設計書のモックアップ（ASCIIアート）を忠実に再現する**見た目だけのコンポーネント**を作成する。
データ・変数は原則べた書き（ハードコード）でよい。
ただし、**画面の表示パターンを切り替えてキャプチャを取得するために必要最低限のロジック**（状態切り替え用の useState 等）は実装してよい。

---

## Step 1: 設計書の確認
1. `docs/designs/screens/screens.md` で対象画面IDを特定する
2. 対応する画面設計書（`docs/designs/screens/s{XX}-*.md`）を読む
3. **モックアップ（ASCIIアート）のUI構成**を正確に把握する
   - 各UI要素の配置・順序・サイズ感
   - テキストラベル・プレースホルダー
   - ボタンの文言
   - **状態ごとのバリエーション（全パターンを把握する）**
     - 例: 初期状態、入力済み状態、エラー表示、ローディング中、データ表示後 等

---

## Step 2: モックコンポーネント作成
1. `docs/guides/guides.md` のディレクトリ構成・コーディング規約に従う
2. モックコンポーネントは `src/features/<機能名>/components/mock/` に配置する
3. **以下のルールでべた書き実装する：**
   - テキスト・ラベル → 文字列リテラル直書き
   - 入力フィールド → 空の `TextField` 等を配置するだけ
   - リストデータ → ダミーの配列をコンポーネント内にハードコード
   - 型定義 → props がなければ不要。最小限でよい
4. **表示パターン切り替え用の最低限ロジックは実装してよい：**
   - ボタン押下で表示パターンを切り替える `useState` / `onClick`
   - エラー表示パターンの確認用にダミーエラー状態の切り替え
   - ローディング表示パターンの確認用
   - **API呼び出し・本番バリデーション等のビジネスロジックは実装しない**
5. MUI コンポーネントを使用し、`sx` prop でスタイリングする
6. 必要に応じて `src/App.tsx` のルーティングや `src/pages/` のページコンポーネントを更新し、画面にアクセスできるようにする

---

## Step 3: ビルド確認
1. `pnpm build` を実行してビルドエラーがないことを確認する
2. エラーがあれば修正して再ビルドする

---

## Step 4: ブラウザ確認・全パターンキャプチャ取得（Playwright MCP使用）
1. `pnpm dev` でローカル開発サーバーを起動する（バックグラウンド実行）
2. Playwright MCP の `browser_navigate` で `http://localhost:5173`（対象パス）を開く
3. **設計書のモックアップに記載された全表示パターンについてキャプチャを取得する：**
   - 初期状態
   - ボタン押下後・入力後など操作後の状態
   - エラー表示状態
   - ローディング表示状態
   - その他、設計書に記載のある表示バリエーション
4. 各パターンごとに `browser_screenshot` → 確認 → 必要なら修正のサイクルを回す
5. 確認OKになったら、キャプチャを以下のパス・命名規則で保存する：
   - **保存先:** `docs/designs/screens/captures/mock`
   - **ファイル名:** `{画面ID}_{連番}_{状態名}_mock.png`
   - 例:
     - `S-01_01_initial_mock.png` — 初期表示
     - `S-01_02_input-filled_mock.png` — 入力済み
     - `S-01_03_validation-error_mock.png` — バリデーションエラー
     - `S-01_04_loading_mock.png` — ローディング中
     - `S-02_01_initial_mock.png` — 学習画面初期表示
6. `browser_close` でブラウザを閉じる

### Playwright MCP コマンド例
```
browser_navigate → url: "http://localhost:5173"
browser_screenshot → (初期状態をキャプチャ)
browser_click → selector: "#some-button"  (表示パターン切り替え)
browser_screenshot → (操作後の状態をキャプチャ)
... 全パターン分繰り返す
browser_close
```

---

## 完了条件
- [ ] 設計書のモックアップに描かれた全UI要素が画面に表示されている
- [ ] ビルドエラーがない（`pnpm build` 成功）
- [ ] 設計書の全表示パターンのキャプチャを Playwright MCP で取得済み
- [ ] キャプチャが `docs/designs/screens/captures/mock/{画面ID}_{連番}_{状態名}_mock.png` に保存されている
- [ ] ビジネスロジック（API呼び出し・本番バリデーション）は未実装であること（表示切り替え用の最低限ロジックのみ許可）
